## 一、项目概述
### 1. 项目背景

随着高校实验教学活动日益丰富，实验室设备种类和使用频率不断增加。传统的设备管理多依赖纸质登记、Excel 表格或口头沟通，存在信息不集中、设备去向不清、归还不及时、损坏责任难追踪、维修记录不完整等问题。为提高实验室资源使用效率和管理规范性，开发面向学生、实验员和管理员的实验室设备借用与报修管理系统。

### 2. 项目目标

- 建立统一的设备信息管理平台
- 实现设备借用申请与审批流程线上化
- 实现借出、归还、逾期、报修等状态的全过程记录
- 支持实验员和管理员进行设备维护、报修处理和公告发布
- 实现基础数据统计，辅助实验室管理决策
- 构建界面清晰、功能完整、可运行演示的信息系统

### 3. 开发环境与条件

- 开发工具：IntelliJ IDEA、Navicat、VS Code、Git
- 前端技术：HTML、CSS、Bootstrap、Thymeleaf
- 后端技术：Spring Boot、MyBatis
- 数据库：MySQL 8.0
- 运行环境：Windows 10/11
- 版本管理：GitHub

## 二、需求分析

### 1. 目标用户与使用场景
- 学生：查看可借设备、提交借用申请、查看审批结果、发起报修申请、查看个人借用 / 报修记录
- 实验员：设备借出 / 归还登记、处理报修、更新设备状态、填写维修说明
- 管理员：用户管理、设备分类管理、借用审批、公告管理、统计分析
- 使用场景：学生借用实验器材、设备损坏报修、管理员统计设备使用情况

### 2. 核心功能清单
- 用户与权限管理：注册登录、角色区分、个人信息维护、密码修改
- 设备信息管理：设备录入、分类管理、状态管理、查询筛选
- 借用申请与审批：申请提交、审批、借出 / 归还登记、借用历史查询
- 报修与维修管理：报修申请、状态更新、维修记录、维修历史查询
- 公告与统计：公告发布、借用 / 报修统计、逾期归还提醒

### 3. 非功能需求
- 性能要求：支持多用户访问，查询与提交响应流畅，无明显卡顿
- 安全要求：权限隔离、登录校验、输入数据校验、数据安全
- 兼容性要求：支持 Chrome、Edge、Firefox 等主流浏览器

## 三、技术路线

### 1. 总体架构
采用 B/S（浏览器 / 服务器）架构。前端使用 Thymeleaf + Bootstrap 构建页面，后端使用 Spring Boot 实现业务逻辑，MyBatis 操作 MySQL 数据库，结构清晰、易于开发与维护

### 2. 主要技术
- 前端：HTML/CSS、Bootstrap、Thymeleaf、JavaScript
- 后端：Spring Boot、Spring MVC、MyBatis
- 数据库：MySQL 8.0
- 协作工具：Git、Markdown

### 3. 数据设计初步考虑
- 主要实体：用户、角色、设备、设备分类、借用申请、借还记录、报修记录、公告
- 关键数据表：
  - user（用户表）、role（角色表）
  - device_category（设备分类表）、device（设备表）
  - borrow_apply（借用申请表）、borrow_record（借还记录表）
  - repair_record（报修记录表）、notice（公告表）
