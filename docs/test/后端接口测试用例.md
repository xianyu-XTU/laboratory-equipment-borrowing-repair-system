# 后端接口测试用例

## 1. 测试环境

| 项目 | 内容 |
|---|---|
| 后端 | Spring Boot 3 |
| 数据库 | MySQL |
| 测试方式 | JUnit + MockMvc + 手工接口测试 |
| 基础地址 | `http://localhost:8080` |

## 2. 测试账号

| 角色 | 用户名 | 密码 |
|---|---|---|
| 管理员 | `admin` | `123456` |
| 实验室管理员 | `lab` | `123456` |

## 4. 手工接口测试用例

### 4.1 登录成功

- 请求：`POST /api/auth/login`
- 请求体：

```json
{
  "username": "admin",
  "password": "123456"
}
```

- 预期：返回 `code=200`，并返回登录用户信息或 token。

### 4.2 登录失败

- 请求：`POST /api/auth/login`
- 请求体：

```json
{
  "username": "admin",
  "password": "wrong-password"
}
```

- 预期：返回失败信息。

### 4.3 查询设备列表

- 请求：`GET /api/devices?page=1&size=10`
- 预期：返回分页设备列表。

### 4.4 新增借用申请

- 请求：`POST /api/borrow/apply`
- 前置条件：学生登录，设备状态为可借。
- 预期：生成待审批借用申请，状态为 `0`。

### 4.5 审批借用申请

- 请求：`POST /api/borrow/approve`
- 前置条件：实验室管理员或管理员登录。
- 预期：审批通过后生成借用记录，设备状态变为已借出。

### 4.6 归还设备

- 请求：`POST /api/borrow/return`
- 前置条件：存在借用中记录。
- 预期：借用记录变为已归还或逾期，设备状态恢复为可借。

### 4.7 提交报修

- 请求：`POST /api/repairs/report`
- 预期：生成报修记录，设备状态变为维修中。

### 4.8 处理报修

- 请求：`POST /api/repairs/handle`
- 前置条件：实验室管理员或管理员登录。
- 预期：维修完成后设备状态恢复为可借。

### 4.9 逾期检测

- 请求：`GET /api/overdue/records`
- 前置条件：存在超过预计归还时间的借用记录。
- 预期：返回逾期记录列表。

### 4.10 统计概览

- 请求：`GET /api/stats/overview`
- 前置条件：实验室管理员或管理员登录。
- 预期：返回用户数、设备数、待审批数、维修数等统计数据。
