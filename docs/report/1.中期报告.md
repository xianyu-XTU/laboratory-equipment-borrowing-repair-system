## 一、项目概述  【中期必填】

### 1. 项目背景

随着高校实验教学活动日益丰富，实验室设备种类和使用频率不断增加。传统的设备管理多依赖纸质登记、Excel 表格或口头沟通，存在信息不集中、设备去向不清、归还不及时、损坏责任难追踪、维修记录不完整等问题。为提高实验室资源使用效率和管理规范性，开发面向学生、实验员和管理员的实验室设备借用与报修管理系统。

### 2. 项目目标

- 建立统一的设备信息管理平台
- 实现设备借用申请与审批流程线上化
- 实现借出、归还、逾期、报修等状态的全过程记录
- 支持实验员和管理员进行设备维护、报修处理和公告发布
- 实现基础数据统计，辅助实验室管理决策
- 构建界面清晰、功能完整、可运行演示的信息系统

### 3. 开发环境与条件

- 开发工具：IntelliJ IDEA、Navicat、VS Code、Git
- 前端技术：HTML、CSS、Bootstrap、Thymeleaf
- 后端技术：Spring Boot、MyBatis
- 数据库：MySQL 8.0
- 运行环境：Windows 10/11
- 版本管理：GitHub

## 二、需求分析  【中期必填】

### 1. 功能需求
本系统主要用于实验室设备管理，面向学生、实验员和管理员三类用户，主要实现设备借用、报修、审批、维护和统计等功能：  

- 学生可以查看可借设备，提交设备借用申请，查看审批结果；在设备损坏时，可以发起报修申请，并查看个人借用记录和报修记录。  

- 实验员可以进行设备借出和归还登记，处理设备报修，更新设备状态，并填写维修说明。  

- 管理员可以进行用户管理、设备分类管理、借用审批、公告管理和统计分析，掌握设备使用和维修情况。  

系统核心功能包括：  

- 用户与权限管理：注册登录、角色区分、个人信息维护、密码修改
- 设备信息管理：设备录入、分类管理、状态管理、查询筛选
- 借用申请与审批：申请提交、审批、借出 / 归还登记、借用历史查询
- 报修与维修管理：报修申请、状态更新、维修记录、维修历史查询
- 公告与统计：公告发布、借用 / 报修统计、逾期归还提醒

主要业务流程包括：学生借用实验器材时，先查看可借设备，然后提交借用申请，由实验员或管理员审批。审批通过后，实验员登记设备借出；设备使用完成后，进行归还登记，并更新设备状态。

设备报修流程为：用户发现设备损坏后提交报修申请，实验员或管理员处理报修，更新维修状态，填写维修说明，维修完成后更新设备状态。

### 2. 非功能需求

- 安全要求：权限隔离、登录校验、输入数据校验、数据安全
- 兼容性要求：支持 Chrome、Edge、Firefox 等主流浏览器

## 三、系统设计  【中期必填】

### 1. 系统架构
本系统采用前后端分离架构，前端负责页面展示和用户交互，后端负责业务逻辑处理，数据库负责数据存储。用户通过浏览器访问系统，前端页面调用后端接口，后端通过数据访问层操作数据库，实现设备管理、借用审批、报修处理、公告发布和统计分析等功能。

系统后端可采用 Spring Boot 构建，使用 Controller 层接收请求，Service 层处理业务逻辑，Mapper 层完成数据库操作，MySQL 用于保存系统数据。

系统总体架构如下：

```text
用户浏览器
   │
   │  页面访问 / 操作请求
   ▼
前端页面
   │
   │  HTTP 请求 / JSON 数据
   ▼
后端系统 Spring Boot
   │
   ├── Controller 控制层：接收请求，返回结果
   │
   ├── Service 业务层：处理借用、审批、报修等业务逻辑
   │
   └── Mapper 数据访问层：操作数据库
   │
   ▼
MySQL 数据库
   │
   └── 存储用户、设备、借用、报修、公告等数据
```

## 四、系统实现  【中期部分填写；结项补全】

### 1. 关键技术
#### 核心框架
- 后端核心框架采用Spring Boot 3 作为后端核心框架，简化 Spring 应用的初始化、配置及部署流程，自动装配依赖、内嵌 Tomcat 服务器，快速搭建稳定的后端服务；结合Spring MVC实现 RESTful 风格接口，完成前后端数据交互。
- 持久层技术使用MyBatis-Plus 替代原生 MyBatis，简化单表 CRUD 操作，无需编写大量 XML 映射文件，内置分页插件、条件构造器，大幅提升数据库操作开发效率；搭配MySQL 8.0 关系型数据库，存储设备信息、用户信息、借用报修记录等业务数据。
- 权限控制基于角色权限拦截实现多角色（学生 / 实验员 / 管理员）访问控制，通过自定义拦截器校验用户登录状态与角色权限，不同角色仅能访问对应功能接口，保证系统数据安全。
工具类与通用技术
- Lombok：通过注解自动生成 get/set、构造方法等代码，减少冗余代码；
- Maven：项目依赖管理与构建工具，统一管理项目 jar 包；
- 前后端分离：前端通过 Ajax/axios 请求后端接口，实现数据异步加载，提升用户体验。

#### 技术难点解决方案
- 多角色权限区分难点：学生、实验员、管理员权限边界模糊，易出现越权访问。解决方案：用户表新增role字段标识角色，登录时存储角色信息到 Session，接口请求前通过拦截器校验角色权限，无权限请求直接拦截。
- 设备借还状态同步难点：设备借用、归还、报修后状态需实时更新，避免重复借用。解决方案：借还 / 报修操作触发数据库事务，原子性更新设备状态（正常 / 借用中 / 维修中），保证数据一致性。
- 分页查询优化难点：设备、报修记录数据量大，原生查询效率低。解决方案：集成 MyBatis-Plus 分页插件，前端传递页码、页大小，后端自动分页返回数据，提升查询速度。

/**
 * 通用返回结果类
 * 统一后端接口返回格式，包含状态码、提示信息、业务数据
 */
public class Result {
    // 状态码：200成功 500失败
    private int code;
    // 返回提示信息
    private String msg;
    // 返回业务数据
    private Object data;

    // 成功静态方法
    public static Result success(Object data) {
        Result result = new Result();
        result.setCode(200);
        result.setMsg("操作成功");
        result.setData(data);
        return result;
    }

    // 失败静态方法
    public static Result error(String msg) {
        Result result = new Result();
        result.setCode(500);
        result.setMsg(msg);
        return result;
    }
}

- 用户登录控制层代码
```java
/**
 * 登录控制器
 * 处理用户登录请求，校验账号密码，保存登录会话信息
 */
@RestController
@RequestMapping("/api/auth")
public class LoginController {

    @Autowired
    private UserService userService;

    /**
     * 用户登录接口
     * @param user 前端传入用户名、密码
     * @param session 会话存储登录用户信息
     * @return 登录结果
     */
    @PostMapping("/login")
    public Result login(@RequestBody User user, HttpSession session) {
        // 根据用户名查询用户
        User loginUser = userService.lambdaQuery()
                .eq(User::getUsername, user.getUsername())
                .one();
        // 校验账号
        if (loginUser == null) {
            return Result.error("账号不存在");
        }
        // 校验密码
        if (!loginUser.getPassword().equals(user.getPassword())) {
            return Result.error("密码错误");
        }
        // 存入会话，保持登录状态
        session.setAttribute("loginUser", loginUser);
        return Result.success(loginUser);
    }
}
```

/**
 * 设备管理业务实现类
 * 实现设备新增、分页查询等基础功能
 */
@Service
public class DeviceServiceImpl implements DeviceService {

    @Autowired
    private DeviceMapper deviceMapper;

    /**
     * 新增实验室设备
     */
    @Override
    public Result addDevice(Device device) {
        // 设置设备默认状态为正常可借用
        device.setStatus("正常");
        int rows = deviceMapper.insert(device);
        return rows > 0 ? Result.success("设备添加成功") : Result.error("设备添加失败");
    }

    /**
     * 分页查询设备列表
     */
    @Override
    public Result getDeviceList(Integer pageNum, Integer pageSize) {
        Page<Device> page = new Page<>(pageNum, pageSize);
        LambdaQueryWrapper<Device> wrapper = new LambdaQueryWrapper<>();
        wrapper.orderByDesc(Device::getCreateTime);
        IPage<Device> devicePage = deviceMapper.selectPage(page, wrapper);
        return Result.success(devicePage);
    }
}

- 设备借用申请业务代码（事务控制）
```java
/**
 * 设备借用业务层
 * 处理学生借用申请，添加事务保证数据一致性
 */
@Service
public class BorrowServiceImpl implements BorrowService {

    @Autowired
    private BorrowMapper borrowMapper;
    @Autowired
    private DeviceMapper deviceMapper;

    /**
     * 提交设备借用申请
     */
    @Override
    @Transactional
    public Result applyBorrow(Borrow borrow) {
        // 设置借用单初始状态
        borrow.setStatus("待审批");
        // 插入借用记录
        int insert = borrowMapper.insert(borrow);
        if (insert <= 0) {
            return Result.error("借用申请提交失败");
        }
        // 更新设备状态为待审批，防止重复申请
        Device device = new Device();
        device.setId(borrow.getDeviceId());
        device.setStatus("待审批");
        deviceMapper.updateById(device);
        return Result.success("借用申请提交成功，请等待管理员审批");
    }
}
```

/**
 * 全局登录拦截器
 * 拦截未登录请求，保护系统接口资源
 */
@Component
public class LoginInterceptor implements HandlerInterceptor {

    @Override
    public boolean preHandle(HttpServletRequest request,
                             HttpServletResponse response,
                             Object handler) throws Exception {
        // 获取会话中的登录用户
        Object loginUser = request.getSession().getAttribute("loginUser");
        // 未登录直接拦截
        if (loginUser == null) {
            response.setContentType("application/json;charset=utf-8");
            response.getWriter().write("{\"code\":500,\"msg\":\"用户未登录，请先登录\"}");
            return false;
        }
        // 已登录放行
        return true;
    }
}
