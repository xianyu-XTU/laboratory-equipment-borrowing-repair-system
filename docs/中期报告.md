## 一、项目概述  【中期必填】

### 1. 项目背景

随着高校实验教学活动日益丰富，实验室设备种类和使用频率不断增加。传统的设备管理多依赖纸质登记、Excel 表格或口头沟通，存在信息不集中、设备去向不清、归还不及时、损坏责任难追踪、维修记录不完整等问题。为提高实验室资源使用效率和管理规范性，开发面向学生、实验员和管理员的实验室设备借用与报修管理系统。

### 2. 项目目标

- 建立统一的设备信息管理平台
- 实现设备借用申请与审批流程线上化
- 实现借出、归还、逾期、报修等状态的全过程记录
- 支持实验员和管理员进行设备维护、报修处理和公告发布
- 实现基础数据统计，辅助实验室管理决策
- 构建界面清晰、功能完整、可运行演示的信息系统

### 3. 开发环境与条件

- 开发工具：IntelliJ IDEA、Navicat、VS Code、Git
- 前端技术：HTML、CSS、Bootstrap、Thymeleaf
- 后端技术：Spring Boot、MyBatis
- 数据库：MySQL 8.0
- 运行环境：Windows 10/11
- 版本管理：GitHub

## 二、需求分析  【中期必填】

### 1. 功能需求
本系统主要用于实验室设备管理，面向学生、实验员和管理员三类用户，主要实现设备借用、报修、审批、维护和统计等功能：  

- 学生可以查看可借设备，提交设备借用申请，查看审批结果；在设备损坏时，可以发起报修申请，并查看个人借用记录和报修记录。  

- 实验员可以进行设备借出和归还登记，处理设备报修，更新设备状态，并填写维修说明。  

- 管理员可以进行用户管理、设备分类管理、借用审批、公告管理和统计分析，掌握设备使用和维修情况。  

系统核心功能包括：  

- 用户与权限管理：注册登录、角色区分、个人信息维护、密码修改
- 设备信息管理：设备录入、分类管理、状态管理、查询筛选
- 借用申请与审批：申请提交、审批、借出 / 归还登记、借用历史查询
- 报修与维修管理：报修申请、状态更新、维修记录、维修历史查询
- 公告与统计：公告发布、借用 / 报修统计、逾期归还提醒

主要业务流程包括：学生借用实验器材时，先查看可借设备，然后提交借用申请，由实验员或管理员审批。审批通过后，实验员登记设备借出；设备使用完成后，进行归还登记，并更新设备状态。

设备报修流程为：用户发现设备损坏后提交报修申请，实验员或管理员处理报修，更新维修状态，填写维修说明，维修完成后更新设备状态。

### 2. 非功能需求
- 性能要求：支持多用户访问，查询与提交响应流畅，无明显卡顿
- 安全要求：权限隔离、登录校验、输入数据校验、数据安全
- 兼容性要求：支持 Chrome、Edge、Firefox 等主流浏览器

> 中期要求：功能清单、主要业务流程、基本非功能需求应明确。

## 三、系统设计  【中期必填】

### 1. 系统架构
本系统采用前后端分离架构，前端负责页面展示和用户交互，后端负责业务逻辑处理，数据库负责数据存储。用户通过浏览器访问系统，前端页面调用后端接口，后端通过数据访问层操作数据库，实现设备管理、借用审批、报修处理、公告发布和统计分析等功能。

系统后端可采用 Spring Boot 构建，使用 Controller 层接收请求，Service 层处理业务逻辑，Mapper 层完成数据库操作，MySQL 用于保存系统数据。

系统总体架构如下：

```text
用户浏览器
   │
   │  页面访问 / 操作请求
   ▼
前端页面
   │
   │  HTTP 请求 / JSON 数据
   ▼
后端系统 Spring Boot
   │
   ├── Controller 控制层：接收请求，返回结果
   │
   ├── Service 业务层：处理借用、审批、报修等业务逻辑
   │
   └── Mapper 数据访问层：操作数据库
   │
   ▼
MySQL 数据库
   │
   └── 存储用户、设备、借用、报修、公告等数据
```

### 2. 模块设计
系统主要包括用户与权限管理、设备信息管理、借用申请与审批、报修与维修管理、公告管理和统计分析等模块。各模块之间相互配合，共同完成实验室设备管理业务。

- 用户与权限管理模块：该模块负责用户注册登录、角色区分、个人信息维护和密码修改。系统根据用户角色分配不同权限，学生、实验员和管理员登录后只能使用对应功能。

- 设备信息管理模块：该模块负责设备录入、分类管理、设备状态管理和设备查询筛选。管理员可以维护设备分类和设备基础信息，实验员可以根据设备使用情况更新设备状态。

- 借用申请与审批模块：该模块负责设备借用申请、审批、借出登记、归还登记和借用历史查询。学生提交借用申请后，由实验员或管理员审批；审批通过后登记设备借出，归还时更新设备状态。
![assets/借用流程（二步式审批流程）图](../assets/借用流程（二步式审批流程）.png)
- 报修与维修管理模块：该模块负责设备报修申请、维修状态更新、维修说明填写和维修历史查询。学生或实验员发现设备故障后提交报修，实验员或管理员处理维修并更新设备状态。
![报修管理流程图](../assets/报修管理流程.png)
- 公告管理模块：该模块负责公告发布、修改、删除和查看。管理员发布实验室通知或设备维护公告，学生和实验员可以查看公告内容。
![公告管理流程图](../assets/公告管理流程.png)
- 统计分析模块：该模块负责统计设备借用情况、报修情况、设备状态分布和逾期归还情况，为管理员了解设备使用情况和管理决策提供参考。

模块关系如下：

```text
用户与权限管理
      │
      ▼
设备信息管理 ── 借用申请与审批 ── 借还记录
      │
      └──────── 报修与维修管理
      │
      ▼
统计分析

公告管理面向所有用户提供通知信息
```

## 四、系统实现  【中期部分填写；结项补全】

### 1. 关键技术
#### 核心框架
- 后端核心框架采用Spring Boot 3 作为后端核心框架，简化 Spring 应用的初始化、配置及部署流程，自动装配依赖、内嵌 Tomcat 服务器，快速搭建稳定的后端服务；结合Spring MVC实现 RESTful 风格接口，完成前后端数据交互。
- 持久层技术使用MyBatis-Plus 替代原生 MyBatis，简化单表 CRUD 操作，无需编写大量 XML 映射文件，内置分页插件、条件构造器，大幅提升数据库操作开发效率；搭配MySQL 8.0 关系型数据库，存储设备信息、用户信息、借用报修记录等业务数据。
- 权限控制基于角色权限拦截实现多角色（学生 / 实验员 / 管理员）访问控制，通过自定义拦截器校验用户登录状态与角色权限，不同角色仅能访问对应功能接口，保证系统数据安全。
工具类与通用技术
- Lombok：通过注解自动生成 get/set、构造方法等代码，减少冗余代码；
- Maven：项目依赖管理与构建工具，统一管理项目 jar 包；
- 前后端分离：前端通过 Ajax/axios 请求后端接口，实现数据异步加载，提升用户体验。

#### 技术难点解决方案
- 多角色权限区分难点：学生、实验员、管理员权限边界模糊，易出现越权访问。解决方案：用户表新增role字段标识角色，登录时存储角色信息到 Session，接口请求前通过拦截器校验角色权限，无权限请求直接拦截。
- 设备借还状态同步难点：设备借用、归还、报修后状态需实时更新，避免重复借用。解决方案：借还 / 报修操作触发数据库事务，原子性更新设备状态（正常 / 借用中 / 维修中），保证数据一致性。
- 分页查询优化难点：设备、报修记录数据量大，原生查询效率低。解决方案：集成 MyBatis-Plus 分页插件，前端传递页码、页大小，后端自动分页返回数据，提升查询速度。
