## 一、项目概述

### 1. 项目背景

随着高校实验教学活动日益丰富，实验室设备种类和使用频率不断增加。传统实验室设备管理方式主要依赖纸质登记、Excel 表格或口头沟通，存在信息不集中、设备去向不清、审批流程不规范、归还不及时、损坏责任难追踪、维修记录不完整等问题。

为提高实验室设备资源利用率和管理规范性，本项目开发了面向学生、实验员和管理员的校园实验室设备管理系统。系统围绕设备信息维护、设备借用申请、借用审批、借出归还、设备报修、维修处理、公告发布和数据统计等核心业务，实现实验室设备管理流程线上化、信息化和可追溯化。

### 2. 项目目标

本项目最终目标是构建一个功能完整、界面清晰、流程闭环、可运行演示的实验室设备管理系统，具体目标如下：

- 建立统一的实验室设备信息管理平台，实现设备基础信息集中维护。
- 实现设备借用申请、审批、借出和归还流程线上化。
- 实现设备状态动态更新，支持正常、待审批、借用中、维修中等状态管理。
- 实现设备报修申请、维修处理和维修记录管理。
- 支持管理员发布系统公告，方便通知实验室相关事项。
- 支持多角色登录与权限控制，区分学生、实验员和管理员操作范围。
- 实现基础统计分析功能，辅助管理员了解设备使用和维修情况。
- 完成系统部署、功能测试、文档整理和项目结项材料归档。

## 二、需求分析

### 1. 功能需求

系统面向学生、实验员和管理员三类用户，围绕实验室设备管理业务实现设备借用、报修、审批、维护、公告和统计等功能。

#### （1）学生端功能

学生用户主要完成设备查看、借用申请、报修申请和个人记录查询等操作：

- 查看可借用设备列表和设备详情。
- 根据实验需要提交设备借用申请。
- 查看个人借用申请审批状态。
- 查看个人借用历史和归还情况。
- 发现设备损坏时提交报修申请。
- 查看个人报修记录及处理状态。
- 查看系统公告和实验室通知。

#### （2）实验员端功能

实验员用户主要负责设备日常管理、借出归还登记和报修处理：

- 查看设备列表、设备详情和设备当前状态。
- 根据审批结果完成设备借出登记。
- 对已归还设备进行归还登记并更新状态。
- 查看设备报修申请。
- 处理设备维修流程，填写维修说明。
- 更新设备运行状态。
- 查看公告和基础统计信息。

#### （3）管理员端功能

管理员用户负责系统整体维护、审批管理、数据统计和公告发布：

- 管理用户信息，包括新增、修改、删除和查询。
- 管理实验室信息和设备分类信息。
- 录入、修改、删除和查询设备信息。
- 审核学生提交的设备借用申请。
- 查看所有借用记录、报修记录和维修记录。
- 发布、修改、删除系统公告。
- 查看设备借用、报修、状态分布等统计信息。
- 对系统基础数据进行维护和检查。

### 2. 核心业务流程

#### （1）设备借用流程

学生借用实验室设备时，先登录系统查看设备列表，选择状态正常且可借用的设备，填写借用时间、用途等信息并提交申请。管理员或实验员审核申请后，若审批通过，则由实验员登记设备借出，系统同步更新设备状态；设备使用完成后，实验员登记归还，系统恢复设备状态并保存借用记录。

```text
学生登录
   │
   ▼
查看可借设备
   │
   ▼
提交借用申请
   │
   ▼
管理员 / 实验员审批
   │
   ├── 审批拒绝：记录拒绝原因，流程结束
   │
   └── 审批通过：登记设备借出
                 │
                 ▼
            设备使用
                 │
                 ▼
            登记归还
                 │
                 ▼
            更新设备状态并保存记录
```

#### （2）设备报修流程

用户发现设备异常后，可提交报修申请。实验员或管理员查看报修记录后进行处理，填写维修说明并更新报修状态。维修完成后，系统将设备状态恢复为正常或标记为其他状态，形成完整维修记录。

```text
用户发现设备故障
   │
   ▼
提交报修申请
   │
   ▼
实验员 / 管理员处理
   │
   ▼
填写维修说明
   │
   ▼
更新报修状态和设备状态
   │
   ▼
形成维修记录
```

## 三、系统设计

### 1. 系统架构

本系统采用前后端分离思想进行设计。前端负责页面展示、用户交互和请求发送；后端负责业务逻辑处理、权限校验和接口响应；数据库负责持久化存储用户、设备、借用、报修、公告等业务数据。

后端系统基于 Spring Boot 构建，采用典型分层架构：Controller 层接收前端请求，Service 层处理业务逻辑，Mapper 层负责数据库访问，MySQL 保存系统数据。

系统总体架构如下：

```text
用户浏览器
   │
   │  页面访问 / 操作请求
   ▼
前端页面
   │
   │  HTTP 请求 / JSON 数据
   ▼
后端系统 Spring Boot
   │
   ├── Controller 控制层：接收请求，返回统一结果
   │
   ├── Service 业务层：处理借用、审批、报修、统计等业务逻辑
   │
   └── Mapper 数据访问层：基于 MyBatis-Plus 操作数据库
   │
   ▼
MySQL 数据库
   │
   └── 存储用户、设备、实验室、借用、报修、公告等数据
```

### 2. 模块设计

系统主要包括用户与权限管理、设备信息管理、借用申请与审批、报修与维修管理、公告管理和统计分析六个核心模块。各模块之间通过用户身份、设备状态和业务记录相互关联，共同构成完整的实验室设备管理流程。

#### （1）用户与权限管理模块

该模块负责用户注册登录、角色区分、个人信息维护和密码修改。系统通过用户角色字段区分学生、实验员和管理员，不同角色登录后进入对应功能页面，并只能访问其权限范围内的接口。

## 四、系统实现

### 1. 关键技术

#### （1）Spring Boot 后端框架

项目后端采用 Spring Boot 3 作为核心框架。Spring Boot 简化了项目初始化、依赖配置和服务启动过程，结合 Spring MVC 提供 RESTful 风格接口，实现前后端数据交互。系统通过 Controller 接收请求，调用 Service 完成业务逻辑，再通过 Mapper 操作数据库。

#### （2）MyBatis-Plus 持久层框架

系统持久层采用 MyBatis-Plus，简化数据库 CRUD 操作。通过 BaseMapper、LambdaQueryWrapper、分页插件等工具，减少重复 SQL 编写，提高开发效率。设备列表、用户列表、借用记录、报修记录等查询均可通过 MyBatis-Plus 快速实现。

#### （3）MySQL 数据库存储

系统使用 MySQL 8.0 存储业务数据，包括用户信息、设备信息、实验室信息、借用记录、报修记录和公告内容。数据库结构围绕设备管理业务设计，支持状态更新和历史记录追踪。

#### （4）角色权限控制

系统通过用户表中的 role 字段区分学生、实验员和管理员。用户登录成功后，将用户信息保存到 Session 中；后续接口请求通过拦截器校验登录状态和角色权限，实现不同角色访问不同功能模块。

#### （5）事务控制与状态同步

设备借用、归还和报修流程会同时影响借用记录、报修记录和设备状态。系统在关键业务方法上使用事务控制，保证相关数据操作要么全部成功，要么全部回滚，避免设备重复借用或状态不一致。

#### （6）统一返回结果

后端接口使用统一 Result 类封装返回结果，包含状态码、提示信息和数据对象，使前端可以统一处理成功、失败和异常结果，提高接口一致性。

### 2. 技术难点及解决方案

| 难点 | 解决方案 |
|---|---|
| 多角色权限边界容易混乱 | 在用户表中设置 role 字段，登录后保存角色信息，通过拦截器和接口逻辑进行权限校验。 |
| 设备状态需随借用、归还、报修同步变化 | 在借用申请、审批、归还、报修处理等关键业务中同步更新设备状态，并使用事务控制。 |
| 列表数据量增加后查询不便 | 集成 MyBatis-Plus 分页插件，前端传递页码和页大小，后端返回分页数据。 |
| 前后端接口返回格式不统一 | 使用 Result 类统一封装 code、msg、data 字段，便于前端统一解析。 |
| 测试过程中出现边界输入问题 | 增加基础参数校验，对空值、错误账号、重复提交等情况进行提示。 |

#### （3）用户登录控制层代码

```java
@RestController
@RequestMapping("/api/auth")
public class LoginController {

    @Autowired
    private UserService userService;

    @PostMapping("/login")
    public Result login(@RequestBody User user, HttpSession session) {
        User loginUser = userService.lambdaQuery()
                .eq(User::getUsername, user.getUsername())
                .one();

        if (loginUser == null) {
            return Result.error("账号不存在");
        }

        if (!loginUser.getPassword().equals(user.getPassword())) {
            return Result.error("密码错误");
        }

        session.setAttribute("loginUser", loginUser);
        return Result.success(loginUser);
    }
}
```

#### （4）设备管理业务层代码

```java
@Service
public class DeviceServiceImpl implements DeviceService {

    @Autowired
    private DeviceMapper deviceMapper;

    @Override
    public Result addDevice(Device device) {
        device.setStatus("正常");
        int rows = deviceMapper.insert(device);
        return rows > 0 ? Result.success("设备添加成功") : Result.error("设备添加失败");
    }

    @Override
    public Result getDeviceList(Integer pageNum, Integer pageSize) {
        Page<Device> page = new Page<>(pageNum, pageSize);
        LambdaQueryWrapper<Device> wrapper = new LambdaQueryWrapper<>();
        wrapper.orderByDesc(Device::getCreateTime);
        IPage<Device> devicePage = deviceMapper.selectPage(page, wrapper);
        return Result.success(devicePage);
    }
}
```

#### （5）设备借用申请业务代码

```java
@Service
public class BorrowServiceImpl implements BorrowService {

    @Autowired
    private BorrowMapper borrowMapper;

    @Autowired
    private DeviceMapper deviceMapper;

    @Override
    @Transactional
    public Result applyBorrow(Borrow borrow) {
        borrow.setStatus("待审批");

        int insert = borrowMapper.insert(borrow);
        if (insert <= 0) {
            return Result.error("借用申请提交失败");
        }

        Device device = new Device();
        device.setId(borrow.getDeviceId());
        device.setStatus("待审批");
        deviceMapper.updateById(device);

        return Result.success("借用申请提交成功，请等待管理员审批");
    }
}
```

#### （6）登录权限拦截器代码

```java
@Component
public class LoginInterceptor implements HandlerInterceptor {

    @Override
    public boolean preHandle(HttpServletRequest request,
                             HttpServletResponse response,
                             Object handler) throws Exception {
        Object loginUser = request.getSession().getAttribute("loginUser");

        if (loginUser == null) {
            response.setContentType("application/json;charset=utf-8");
            response.getWriter().write("{\"code\":500,\"msg\":\"用户未登录，请先登录\"}");
            return false;
        }

        return true;
    }
}
```

## 六、用户手册

### 1. 安装部署说明

#### （1）环境准备

运行本系统前需准备以下环境：

- JDK 17 及以上版本
- Maven 3.6 及以上版本
- MySQL 8.0
- Git
- Chrome / Edge 等浏览器
- IntelliJ IDEA 或 VS Code

#### （2）获取项目代码

```bash
git clone https://github.com/xianyu-XTU/lab-equipment-management-system.git
cd lab-equipment-management-system
```

#### （3）初始化数据库

1. 打开 MySQL 或 Navicat。
2. 新建数据库 `lab_equipment`。
3. 执行项目 `database/lab_equipment.sql` 文件，导入系统表结构和基础数据。
4. 检查用户表、设备表、借用表、报修表等是否创建成功。

#### （4）修改后端配置

进入后端配置文件：

```text
backend/src/main/resources/application.yml
```

根据本机 MySQL 环境修改数据库账号和密码：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/lab_equipment?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: root
    password: root
```

#### （5）启动后端服务

进入后端目录，执行 Maven 启动命令：

```bash
cd backend
mvn spring-boot:run
```

后端启动成功后，浏览器或前端页面即可通过后端接口访问系统功能。

#### （6）启动前端页面

进入前端目录，根据项目实际页面入口打开登录页面，或通过本地服务器运行前端项目。若使用 VS Code，可使用 Live Server 打开前端页面进行访问。

```text
frontend/
```

### 3. 操作指南

#### （1）登录系统

用户打开登录页面，输入用户名和密码后点击登录。系统验证账号密码，登录成功后根据用户角色进入对应页面；若账号不存在或密码错误，系统给出提示。

#### （2）学生操作指南

学生登录后可进行以下操作：

1. 在设备列表中查看可借用设备。
2. 点击设备详情查看设备信息。
3. 选择需要借用的设备，填写借用申请并提交。
4. 在个人借用记录中查看审批状态。
5. 设备损坏时进入报修页面，填写故障描述并提交报修申请。
6. 在报修记录中查看维修处理结果。
7. 查看系统公告，了解实验室通知。

#### （3）实验员操作指南

实验员登录后可进行以下操作：

1. 查看设备列表和设备当前状态。
2. 对已审批通过的借用申请进行借出登记。
3. 对归还设备进行归还登记。
4. 查看设备报修申请。
5. 处理报修记录，填写维修说明。
6. 更新设备状态。
7. 查看公告和相关统计数据。

#### （4）管理员操作指南

管理员登录后可进行以下操作：

1. 进入用户管理页面，新增、修改、删除和查询用户。
2. 进入设备管理页面，维护设备信息、分类和状态。
3. 进入借用管理页面，审核学生借用申请。
4. 进入报修管理页面，查看和处理设备故障信息。
5. 进入公告管理页面，发布、修改或删除公告。
6. 进入统计分析页面，查看设备状态、借用数量和报修情况。
7. 定期检查系统数据，确保设备信息和业务记录准确。

### 2. 不足与改进方向

虽然系统已完成主要功能，但仍存在一些可以继续优化的方向：

- 权限控制目前以基础角色校验为主，后续可进一步细化到按钮级和接口级权限。
- 密码存储仍可进一步优化，后续可引入加密存储机制，提高账号安全性。
- 统计分析目前以基础数据展示为主，后续可增加图表展示和多维度筛选。
- 前端页面可继续优化交互体验，例如增加加载提示、批量操作和更友好的表单校验。
- 系统部署目前以本地运行为主，后续可部署到服务器并配置 Nginx、Docker 等环境。
- 可增加操作日志和异常日志，便于管理员追踪系统操作和排查问题。
- 可增加导出功能，例如导出设备清单、借用记录和维修记录。
